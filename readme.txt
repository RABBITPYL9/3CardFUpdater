stopmodule.py - модуль остановки процессинга 3CardF
startmodule.py - модуль запуска процессинга 3CardF
updater.py - скрипт обновления модулей
log.txt - лог работы скрипта по обновлению

Инструкция по запуску:
Скрипт проверяет выданные обновления от разработчиков на сетевом ресурсе
строка под номером 22(path = "C:\\Release\\") в файле updater.exe
При нахождении выданных релизов модулей - идет обновлениие, в директорию указанную на по адресу C:\\UC3D\EXE220 каждого модуля
Перед проверкой обновления, подключается модуль stopmodule.py который останавливает процессинговый центр, после него идет обновление, после обновления - запускает процессинговый центр
скрипт обновления(updater.py) необходимо запускать каждый день, в конце рабочего дня, например до 23:55 дня, так как выдавая релиз, разработчик указывает дату в названии папки
Пример каталога с релизом:
S:\220_38\20210422_220_38\GateWay
Где S: - диск сетевого ресурса
Где 220_38 - версия процессинга
Где 20210422_220_38 - дата с указанием версии

p.s если возникнут ошибки обновления или будет найден неизвестный модуль, то в логе работы скрипта об этом будет запись, с указанием шага обновления, времени до секунды
